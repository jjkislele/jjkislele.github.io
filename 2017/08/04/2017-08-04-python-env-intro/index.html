Python环境的搭建

wp intro python
 2017/08/04

最最重要的话：千千万万不要去搜某个软件然后在某软件“中心”一键下载。

一是怕这个软件“中心”会给你加点其它你意想不到的“全家桶”；二是你控制不了版本，书上要你用2.x还是3.x，这是有区别的，书里面提到的包，有可能在3.x里能运行，在2.x就不能运行了。这样下载真心很，不要这么搞。

一、下载与安装

1.1 哪里的安装包

还是用某度搜下python，这么大的org标识，python.org点进去好不好。

进入页面后点download直接到下载页，2.x和3.x随你下载啊。

if you want to use a specific third party package or utility that doesn’t yet have a released version that is compatible with Python 3, and porting that package is a non-trivial task, you may choose to use Python 2 in order to retain access to that package.

在这个页面往下拉就可以下载到更加“精确”的版本，例如Python 2.7.13，Python 3.5.1等等等等。

正确下载好后，双击安装就可以了。

1.2 环境变量

还有一步，添加环境变量，意思就是告诉系统你装了python。或许在低版本程序里你需要做这样的动作，我没弄明白这里的python需不需要添加，不过还是加上，以前在没加的时候安装第三方包好像会出错（似乎是npm）。

“控制面板\所有控制面板项\系统”的高级系统设置里面的环境变量

在PATH里面加入你安装的python的目录就好，注意在PATH变量值的后面添加，并用英文;隔开。

第一章就结束了，这个时候你的python应该就已经ready to rock了，不过python强大之处，不仅在于它简练的语法，探索式的编程方式，还在于它开放的无穷无尽的第三方库 third-party library ，用于数据处理的什么panda啊，scipy啊，scikt-learn啊，这些都是我以前使用过的包，实用。

二、第三方包的安装

由于城墙的缘故，用ez-install或者pip很容易超时，所以就必须需要国内镜像仓库，或者在什么git啊SourceForge啊把源码包下载下来自己在本地装。

2.1 pip+国内镜像

http://pypi.douban.com/simple/ 豆瓣
http://mirrors.aliyun.com/pypi/simple/ 阿里
http://pypi.hustunique.com/simple/ 华中理工大学
http://pypi.sdutlinux.org/simple/ 山东理工大学
http://pypi.mirrors.ustc.edu.cn/simple/ 中国科学技术大学
https://pypi.tuna.tsinghua.edu.cn/simple 清华

2.2 pip+本地安装

直接 pip -install 包名就好了。
