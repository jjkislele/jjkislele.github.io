电脑蓝屏了，引导坏了

过了很久很久，我的第一个笔记本——vestro 5460——在我卸下已经鼓包的电池不知道多久之后，终于，它的bios电池也没电了。也是巧合，我突然想找一个以前硬盘上的文件，于是欣然打开它，一来就开始报错，提示bios电池用尽，时间信息丢失，f1跳过设置、跳过自检，然后终于出现了熟悉的戴尔logo，耐心等待片刻后，它，蓝屏了，报错“inaccessible boot device”。

“inaccessible boot device”，难道我的硬盘坏了？不应该，坏了就过不了启动的自检，难道是坏道，刚好把有boot信息的扇区抹掉了？冷汗之余，赶紧用戴尔自带的自检程序跑一遍，除了提示我没电池外，看任务log也没看出什么来，似乎我的硬件是正常工作的。

百度的结果基本上就是重装电脑，谷歌也是同样的结果，痛心无奈下，只好睡觉冷静。过了一天，思来想去，想起唯一的可能，就是boot引导坏了，得用pe上的工具修复，但目前国内的那些pe都不大干净，ghost居然还给你夹带点其它东西，看教程又看见有bcdboot这样的工具，那么就试试吧！

需要打开终端，我蓝屏额，怎么开终端？事实上，windows8在恢复模式下，是自带终端的，在开机界面狂按f8，找到“命令提示符”字样的选项，奇迹地，把终端给弄出来了，接着就是修复efi引导了。

修复efi引导，网上还是有教程的，大概把过程摘录如下，免得后面又出现相同状况让人自闭：

  1. diskpart, 打开工具
  2. list disk, 查看盘符
  3. sel disk 0, 选择盘符，这里的0，也就是第一个盘
  4. list par, 查看扇区
  5. sel par 1，选择扇区，应该是第一个，为啥编号从1开始而不是0，我也不知道
  6. assign letter=o, 将这个引导扇区标记为o
  7. exit, 退出diskpart工具
  8. bcdboot c:\windows /s o: /f uefi /l zh-cn

成功了，就重启吧，应该能进系统了。

另外，因为电脑的bios电池干了，于是拆机换电池，应该用cr2016，我用的是cr2032，16代表高度16mm，高了一倍，还好塞得进去，顺带一提，我之前是修过电脑的，居然竟然，我的半高ssd被拆了，不知道是商家忘了，还是真的无良，那只是一张24GB的半高ssd啊，是拿来加速hd的，根本不值钱，唉，难过。下次修电脑，一定要检查好硬件是否被替换。
